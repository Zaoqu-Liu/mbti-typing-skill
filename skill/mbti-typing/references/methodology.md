## Lessons From Deep Dialogue Analysis

The strongest local case study covered many rounds, hundreds of messages, and multiple independent review passes. The useful lesson is not the final label. The useful lesson is that accurate typing emerged only after repeated hypothesis failure.

Key lessons:

1. **Start from a claim, not a conclusion.** A third-party or self-test type is evidence, but it is not the answer.
2. **Surface similarity is dangerous.** A turbulent, reflective, emotionally reactive executive type can look like INFP if the interviewer tracks sensitivity instead of function order and default cognition.
3. **Adjacent types can share almost everything.** ENTJ and INTJ use the same four functions in a different order. Evidence for Ni, Te, Fi, and Se alone cannot separate them; the interview must test the default starting point and energy metabolism.
4. **The best question may appear late.** "When you are free and alone, what does your mind default to?" can separate Te-dominant external optimization from Ni-dominant private modeling better than generic E/I questions.
5. **User corrections are diagnostic gold.** When the subject says an interpretation is too literary, too existential, or already solved in real life, update the model. Do not defend the narrative.
6. **Framework mixing must be labeled.** A/T, Big Five, Enneagram, attachment, culture, and cognitive functions can enrich a report, but each has a different evidentiary status.
7. **Precise-looking scores are often fake precision.** Function "95/85/65/40" can be useful as a sketch, but should not be framed as measured data without an instrument.
8. **A final answer must include a serious runner-up.** If no runner-up can be steelmanned, the analysis is probably confirming rather than diagnosing.

## Diagnostic Protocol

### Phase 1: Intake

Capture:

- Current type claim and source.
- Prior test results and whether the instrument is known.
- The specific dispute: whole type, E/I, function order, A/T, or report quality.
- Context: age, work role, stress level, culture, life stage, and relationship situation if relevant.

For file analysis:

- Read summary files first.
- Search for "反转", "证伪", "修正", "误判", "置信", "评审", "不能解释", and "用户修正".
- Sample raw turns around every major reversal and every user correction.
- Avoid copying sensitive content into the reusable report unless the user asks and it is necessary.

### Phase 2: Candidate Set

Create 3-6 active candidates:

- Claimed type.
- Main adjacent type sharing the strongest features.
- Type that explains the same surface through a different mechanism.
- Type suggested by stress pattern.
- Type suggested by public role or career if it could be role contamination.

Example candidate logic:

- INFP claim with strong execution: compare INFP, INTJ, ENTJ, ENFP, ISFP, INFJ.
- ENTJ vs INTJ: both Te/Ni/Se/Fi; test Te-dom vs Ni-dom and Se tertiary vs Fi tertiary.
- ESTJ vs ENTJ: both Te-forward; test Si precedent/stability vs Ni future-pattern compression.

### Phase 3: Coarse Function Triage

Ask scenarios that identify default cognition:

- How the person enters a messy problem.
- How they decide under ambiguity.
- What kind of information feels actionable.
- What they do under criticism.
- What breaks under stress.

Do not over-trust self-description such as "I am logical" or "I am emotional". Track process.

### Phase 4: Conflict Drill

After the first round, find contradictions:

- Se looks both strong and weak.
- Fi looks central in conflict but absent in daily life.
- E looks high in leadership but low in social leisure.
- Ni looks strong but may be Te speed described as intuition.

Turn each contradiction into a follow-up. A contradiction is often the most valuable diagnostic object.

### Phase 5: Adjacent Pair Duel

When two types remain, stop broad questioning. Ask only pair-specific discriminators.

For each discriminator:

- Define what each type would usually do.
- Ask for a concrete repeated pattern.
- Ask for a counterexample.
- Ask what the behavior costs or restores.

Close only after the runner-up has at least two evidence points it struggles to explain.

### Phase 6: Cross-Framework Check

Use Big Five to check broad trait plausibility:

- Extraversion: social energy, assertiveness, stimulation, positive affect.
- Conscientiousness: order, self-discipline, achievement striving.
- Agreeableness: compassion, compliance, conflict avoidance.
- Neuroticism: volatility, anxiety, rumination, threat sensitivity.
- Openness: abstraction, novelty, aesthetics, intellectual curiosity.

Use other frameworks as explanatory lenses:

- Attachment can explain emotional deactivation or protest.
- Culture can explain male stoicism, family responsibility, authority behavior, and money pressure.
- Life stage can explain instability or identity reconstruction.
- Role demands can mimic type, especially leadership, medicine, academia, entrepreneurship, and management.

### Phase 7: Final Synthesis

Write the conclusion as:

1. Leading formulation.
2. Runner-up formulation.
3. Evidence that separates them.
4. Evidence that remains ambiguous.
5. Falsifiers that would change the answer.
6. Growth or practical implications, if requested.

## Evidence Weighting

Use likelihood ratios as a thinking aid:

| LR | Meaning |
|---:|---|
| 10+ | Strong support; use rarely |
| 5 | Moderate support |
| 2 | Weak support |
| 1 | Neutral |
| 0.5 | Weak contradiction |
| 0.2 | Moderate contradiction |
| 0.1 or lower | Strong contradiction; use rarely |

Rules:

- Reduce weight for correlated questions.
- Reduce weight for socially desirable answers.
- Increase weight for repeated real behavior across settings.
- Increase weight for spontaneous corrections.
- Increase weight for cost signatures: what drains, restores, scares, embarrasses, or feels unbearable.
- Do not multiply many weak signs as if independent.

## Contradiction Gates

Before finalizing, answer:

- What evidence would the leading type fail to explain?
- What evidence would the runner-up explain better?
- Which answers may be role, culture, stress, or maturity rather than type?
- Which interpretation came from the subject's words, and which came from the analyst's theory?
- Have all non-standard terms been labeled as observations rather than accepted theory?

## Report Writing

Use this structure:

- "Best-supported type": one paragraph.
- "Why not the main alternatives": table.
- "Function/process evidence": table with observations.
- "Trait cross-check": Big Five sketch.
- "Uncertainty and falsifiers": explicit section.
- "Practical implications": optional, grounded in the person's stated goals.

Prefer:

- "The evidence supports Te-Ni as the dominant process."
- "INTJ remains a live runner-up because..."
- "This is a working formulation from interview data."

Avoid:

- "Proven 99.7%" unless explicitly framed as an internal heuristic from a toy Bayesian update.
- "This behavior means Fi is weak" when attachment, culture, and learned coping could also explain it.
- "All ENTJs..." or "INTJs never..."

## Failure Modes

- **Barnum portrait:** Beautiful but not falsifiable.
- **Label laundering:** Infer type from behavior, then use the type to explain the same behavior.
- **Single-axis lock:** Treat E/I or T/F as decisive while ignoring function order.
- **Framework soup:** Mix MBTI, 16Personalities, Big Five, Enneagram, and attachment without boundaries.
- **Over-literary inference:** Turn ordinary tiredness into existential crisis without subject confirmation.
- **Expert-prior overweighting:** Give a "small expert" or online test too much authority.
- **Self-report capture:** Let the subject's desired identity dominate real behavior evidence.
